# Чтение и обработка Excel

Шаблон для чтения Excel файла с обработкой данных и единым форматом результата.

Поддерживает работу как с одним листом, так и со всей книгой.

---

## Назначение

Используйте данный шаблон, если вам нужно:

- прочитать данные из Excel файла  
- обработать данные в табличном виде  
- получить структуру колонок  
- работать с одним или несколькими листами  
- стандартизировать обработку Excel  

---

## Входные параметры

Функция принимает параметры для управления чтением Excel:

| Параметр        | Тип    | Обязательный | Описание                                         |
| --------------- | ------ | ------------ | ------------------------------------------------ |
| `file_path`     | string | да           | Путь к Excel файлу                               |
| `sheet_name`    | string | нет          | Имя листа (если не указано — читается вся книга) |
| `dtype`         | dict   | нет          | Типы данных колонок                              |
| `names_columns` | list   | нет          | Список колонок для чтения                        |
| `skiprows`      | int    | нет          | Количество пропускаемых строк                    |
| `nrows`         | int    | нет          | Количество читаемых строк                        |


- `file_path` — полный путь к Excel файлу 
- `sheet_name` — имя листа; если не задан, будет прочитана вся книга
- `dtype` — позволяет явно задать типы колонок
- `names_columns` — используется для выборочного чтения колонок
- `skiprows` — пропуск строк в начале файла 
- `nrows` — ограничение количества строк 

### Особенность поведения
если `sheet_name` указан → режим **single_sheet**
если `sheet_name` не указан → режим **multi_sheet** (возвращается словарь DataFrame по листам)

### Рекомендуется:

- явно указывать `sheet_name`, если нужен конкретный лист
- использовать `nrows` при работе с большими файлами
- задавать `names_columns` для ускорения чтения
- использовать `dtype` для контроля типов данных

---

## Структура шаблона

Шаблон состоит из следующих частей:

### 1. Инициализация результата

Создаётся объект `execution_result` со стандартной структурой.

---

### 2. Проверки входных данных

Выполняются базовые проверки:

- существует ли файл  
- поддерживается ли формат файла  
- существует ли указанный лист  

---

### 3. Чтение Excel

Данные читаются с помощью блока чтения Excel.

Поведение зависит от параметра `sheet_name`:

- если указан — читается один лист  
- если не указан — читается вся книга  

---

### 4. Определение режима работы

После чтения определяется режим:

- `single_sheet` — один лист  
- `multi_sheet` — вся книга  

---

### 5. Обработка данных

#### В режиме `single_sheet`:

- определяется количество строк  
- извлекаются названия колонок  
- возвращаются данные  

#### В режиме `multi_sheet`:

- данные разбиваются по листам  
- для каждого листа определяется:
  - количество строк  
  - список колонок  
  - данные  

---

### 6. Сохранение результата

Результат записывается в `execution_result` в зависимости от режима.

---

### 7. Обработка ошибок

При возникновении ошибки:

- устанавливается статус `error`  
- формируется сообщение  
- сохраняется описание ошибки  

---

### 8. Возврат результата

Функция всегда возвращает `execution_result`.

---

## Возвращаемое значение

Функция возвращает словарь `execution_result`.

### Структура:

#### Режим `single_sheet`

```json
{
  "status": "success | error",
  "message": "string",
  "data": {
    "mode": "single_sheet",
    "rows_count": 0,
    "columns": [],
    "full_data": "DataFrame"
  },
  "error_description": "string"
}
```

#### Режим `multi_sheet`

```json
{
  "status": "success | error",
  "message": "string",
  "data": {
    "mode": "multi_sheet",
    "sheets" {
      "Sheet1": {
        "rows_count": 0,
        "columns": [],
        "full_data": "DataFrame"
      }
    }
  },
  "error_description": "string"
}
```

## Особенности

- `full_data` возвращается как **DataFrame**, а не список
- в режиме `multi_sheet` данные возвращаются по каждому листу отдельно
- шаблон не выполняет сложную бизнес-обработку — только базовую подготовку данных
- шаблон можно расширять дополнительной логикой

## Когда использовать

- обработка Excel файлов
- загрузка данных из таблиц
- подготовка данных для дальнейшей обработки
- интеграции с файлами


## Когда не использовать
- простые вычисления без Excel
- случаи, где данные уже находятся в памяти
- когда требуется только запись в Excel

## Рекомендации по использованию
- не возвращайте полный DataFrame для очень больших файлов без необходимости
- используйте `nrows` для ограничения объёма данных
- обрабатывайте результат в отдельных функциях
- комбинируйте с шаблоном Retry для нестабильных источников

## Примеры использования
- чтение отчёта из Excel
- загрузка данных для обработки
- агрегация данных по колонкам
- подготовка данных для API
