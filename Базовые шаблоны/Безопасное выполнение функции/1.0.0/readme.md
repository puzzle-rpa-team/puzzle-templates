# Выполнение функции с обработкой ошибок и логированием

Шаблон для создания функций с единым контрактом результата, обработкой ошибок и логированием.

Позволяет стандартизировать разработку и упростить переиспользование функций в процессах.

---

## Назначение

Используйте данный шаблон, если вам нужно:

- реализовать функцию с гарантированным возвратом результата  
- обрабатывать ошибки без прерывания выполнения  
- логировать ошибки  
- передавать результат в унифицированном формате  

---

## Входные параметры

Функция может принимать входные параметры, необходимые для выполнения логики.

Параметры передаются в функцию при вызове и используются в основной логике.

Рекомендуется:

- явно задавать входные параметры функции  
- использовать понятные имена переменных  
- не использовать глобальные переменные  

Примеры входных параметров:

- путь к файлу (`file_path`)
- URL (`url`)
- данные для обработки (`input_data`)

---

## Структура шаблона

Шаблон состоит из 5 основных частей:

1. **Инициализация результата**

    В переменной execution_result сохраняется результат выполнения функции.

    Стандартная структура включает:
    - status — статус выполнения (success / error)
    - message — текстовое описание результата или ошибки
    - data — результат работы функции

    При необходимости можно добавлять дополнительные поля, но рекомендуется сохранять базовую структуру для единообразия.

2. **Основная логика**

    Здесь реализуется ключевая обработка данных:
    получение, преобразование, взаимодействие с системами и т.д.

    Рекомендуется:

    - минимизировать побочные эффекты
    - использовать понятные промежуточные переменные
    - обрабатывать критичные проверки до записи результата

3. **Сохранение результата**

    Сохраняем результат выполнения функции в поле data.

    В data рекомендуется возвращать:
    - обработанные данные
    - итоговые значения
    - структуры, пригодные для дальнейшего использования

4. **Обработка ошибки**

    При возникновении исключения:
    - устанавливается статус error
    - формируется понятное сообщение
    - сохраняется описание ошибки

    **Важно**: функция не должна "падать" — ошибка должна быть возвращена в результате.

5. **Возврат результата**

    Независимо от исхода выполнения функция должна вернуть execution_result, содержащий:
    - статус выполнения
    - сообщение
    - результат 
    - описание ошибки

---

## Возвращаемое значение (`execution_result`)

Функция всегда возвращает словарь **`execution_result`**.

### Рекомендуемая структура:

```json
{
  "status": "success | error",
  "message": "string",
  "data": {},
  "error_description": ""
}
