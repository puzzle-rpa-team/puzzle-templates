# HTTP запрос

Шаблон для выполнения HTTP-запроса с поддержкой повторных попыток (retry) и единым форматом результата.

Позволяет выполнять устойчивые интеграции с API и обрабатывать ответы в стандартизированном виде.

---

## Назначение

Используйте данный шаблон, если вам нужно:

* выполнить HTTP-запрос к API
* обработать ответ сервера
* повторить запрос при временных ошибках
* унифицировать работу с внешними сервисами

---

## Входные параметры

Функция принимает параметры для выполнения HTTP-запроса:

| Параметр      | Тип       | Обязательный | Описание                        |
| ------------- | --------- | ------------ | ------------------------------- |
| `url`         | string    | да           | URL запроса                     |
| `headers`     | dict      | нет          | Заголовки запроса               |
| `params`      | dict      | нет          | Query-параметры                 |
| `data`        | dict      | нет          | Данные запроса в виде словаря   |
| `max_retries` | int       | да           | Максимальное количество попыток |
| `retry_delay` | int/float | да           | Задержка между попытками (сек)  |

---

* `url` — адрес API
* `headers` — используется для передачи токенов, авторизации и служебных данных
* `params` — параметры запроса (обычно для GET)
* `data` — Данные запроса в виде словаря
* `max_retries` — количество попыток выполнения запроса
* `retry_delay` — задержка между попытками

---

### Особенности настройки блока

Параметры HTTP-запроса настраиваются непосредственно в блоке:

* HTTP-метод (GET, POST, PUT и др.)
* таймаут
* кодировка
* тип аутентификации
* проверка SSL-сертификата

---

## Структура шаблона

Шаблон состоит из следующих частей:

### 1. Инициализация результата

Создаётся объект `execution_result` со стандартной структурой.

---

### 2. Проверка входных параметров

Проверяется корректность входных данных:

* указан ли URL

---

### 3. Цикл повторных попыток

Выполняется цикл до `max_retries`, внутри которого:

* выполняется HTTP-запрос
* обрабатывается результат
* при ошибке выполняется повтор

---

### 4. Выполнение HTTP-запроса

Запрос выполняется с использованием блока HTTP.

Ответ возвращается в виде:

* результат (`response_result`)
* заголовки (`headers`)
* код ответа (`status_code`)

---

### 5. Проверка ответа

Если код ответа не входит в диапазон `2xx`, считается ошибкой:

* генерируется исключение
* попытка считается неудачной
* выполняется повтор

---

### 6. Сохранение результата

При успешном выполнении:

* устанавливается `status = success`
* сохраняется ответ
* фиксируется количество использованных попыток

---

### 7. Обработка ошибок

При ошибке:

* сохраняется описание ошибки
* добавляется запись в `errors`
* выполняется задержка перед повтором

---

### 8. Возврат результата

Функция всегда возвращает `execution_result`.

---

## Возвращаемое значение

Функция возвращает словарь `execution_result`.

### Структура:

```json
{
  "status": "success | error",
  "message": "string",
  "data": {
    "attempts_used": 0,
    "status_code": 0,
    "response_result": "",
    "headers": {},
    "errors": []
  },
  "error_description": "string"
}
```

---

## Особенности

* успешным считается любой HTTP-код из диапазона `2xx`
* при неуспешных кодах (`4xx`, `5xx`) выполняется повтор
* сохраняется история ошибок (`errors`)
* при неудаче возвращается последний полученный ответ (если он был)
* `response_result` автоматически преобразуется в зависимости от ответа сервера

---

## Когда использовать

* интеграции с API
* получение данных из внешних сервисов
* отправка данных (POST/PUT)
* взаимодействие с REST-сервисами

---

## Когда не использовать

* простые локальные операции
* работа без сетевых вызовов
* сценарии без необходимости повторных попыток

---

## Рекомендации по использованию

* используйте разумное количество попыток (обычно 2–5)
* добавляйте задержку при работе с нестабильными сервисами
* проверяйте `status_code` при обработке результата
* логируйте ошибки для анализа

---

## Примеры использования

* получение данных из API
* отправка форм и JSON-запросов
* интеграция с внутренними сервисами
